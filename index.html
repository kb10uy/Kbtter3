Kbtterシリーズ、ついに第3世代へ。

2014年3月に公開された初代Kbtter。4月に制作が開始され半月経たずに開発終了になったKbtter2 Polyethylene。
その根底にある何かを受け継ぎ、2014年5月、Kbtter3 Polystyreneが誕生しました。

後追いの宿命

Kbtter2から受け継がれた基本方針、それは
　「知る限りの全てのTwitterクライアントで決着を付ける」　
です。後発のクライアントである分、従来のクライアントのいいところを多く取り込んでいます。 参考にしているクライアントの選択は開発者の独断と偏見ですが。

  • Shrimp
    • ulicknormanowen氏が開発。
      タブをメインにした画面構成は、これに影響されています。
      他にもUIの細かいところを参考にさせていただきました。
  • Soarer
    • java_shit氏が開発。 最初に知った時に感動しました。初代と3でレイアウトが大きく違うのは、これの存在によるものです。
      シンプルイズベストな何かを感じられました。
  • mikutter
    • toshi_a氏が開発。 言わずと知れたておくれクライアント。ツイートの表示はこれを参考にしています。
      実質的に画面はこれが一番近いかもしれない。

その他いろいろなものを参考にして作られています。ありがとうございます。

Kbtter3 主な機能

今までのKbtterシリーズの機能も受け継いでいます。

  • Streaming API対応
    • 今となってはほぼ業界(?)標準のStreaming API。Kbtter3はもちろん対応。
  • プラグイン機能
    • IronPythonやLuaを使ってプラグインを作成・使用することが出来ます。 用途としては、指定条件ツイートの収集、update_name、あんふぁぼ反応とか？
  • DM閲覧・送受信
    • 意外と対応しているクライアントが少なさそうなDM機能。
      Kbtter3には、あります。
  • ツイートのブックマーク
    • お気に入りとは別に、保存しておきたいと思ったツイートをブックマークとして 登録する機能を実装。お気に入りよりも便利に検索・整理が出来ます。

動作環境

性能的には、Windows Vista後期以降のモデルなら快適に動作すると思います。

  • Windows 7以降
  • .NET Framework 4.5以降
  • 1GB以上のRAM(2GB以上推奨)
  • 2GHz以上くらいのCPU
  • 256MB以上のVRAM
